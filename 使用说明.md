## 插件概述

**orca-import-export** 是一款为 Orca Note 设计的多格式导入导出插件，支持在 Orca Note 与其他笔记应用（Logseq、Obsidian、思源笔记等）之间进行数据迁移，同时支持多种文档格式（Markdown、HTML、Word、JSON、OPML）的导入导出。

### 核心特性

- ✅ **多格式互转**：支持 Logseq / Obsidian / Orca Note / SiYuan 四种笔记应用格式，以及 Markdown、HTML、Word、JSON、OPML、TXT 六种文件格式
- ✅ **富文本双向转换**：支持高亮、加粗、斜体、删除线等富文本样式的导入和导出
- ✅ **挖空模式**：支持多种挖空语法（如 `〖内容〗`、`[[c1::xx]]`）与 Orca 挖空效果的双向转换
- ✅ **表格支持**：保持表格结构在导入导出过程中的完整性
- ✅ **图片支持**：支持 Markdown 图片语法 `![](url)` 的导入和导出
- ✅ **进度反馈**：导入导出过程中显示实时进度，支持取消操作
- ✅ **实时预览**：导入前可预览转换结果，支持编辑调整

## 安装与卸载

### 安装

1. 下载最新版本的插件包（`.zip` 文件）
2. 在 Orca Note 中打开 **设置 > 插件**
3. 点击 **安装插件**，选择下载的 `.zip` 文件
4. 插件自动加载，工具栏会出现导入/导出按钮

### 卸载

1. 在 Orca Note 中打开 **设置 > 插件**
2. 找到 **orca-import-export**，点击右侧的 **卸载**
3. 插件自动清理并卸载

## 快速开始

### 导入文件

1. 在工具栏点击 **导入** 按钮，或使用快捷键 `Ctrl+Shift+I`
2. 选择要导入的文件（支持 `.md`、`.txt`、`.opml`、`.xml`、`.json`）
3. 在预览面板中调整导入设置（格式、挖空模式、富文本转换等）
4. 点击 **导入**，选择插入位置（子块或同级）
5. 等待进度完成，内容自动插入到当前文档

### 导出文件

1. 在工具栏点击 **导出** 按钮，或使用快捷键 `Ctrl+Shift+E`
2. 选择导出格式（如 Logseq 样式、Markdown、HTML 等）
3. 调整导出设置（最大深度、富文本转换、挖空模式等）
4. 点击 **导出**，文件自动下载

## 导入功能

### 导入方式

插件支持三种导入方式：

| 方式 | 说明 | 操作 |
|------|------|------|
| **文件选择** | 点击"选择文件"按钮 | 点击导入对话框中的文件选择按钮 |
| **拖放导入** | 将文件拖放到对话框的拖放区域 | 从文件管理器拖放文件到对话框 |
| **剪贴板粘贴** | 粘贴剪贴板中的文本内容 | 使用 `Ctrl+V` 粘贴到对话框的源码编辑器 |

### 导入设置

导入对话框提供以下设置选项：

#### 基础设置

- **导入格式**：自动检测或手动选择源格式（Logseq、Obsidian、Orca、SiYuan、纯文本）
- **插入位置**：作为子块插入（推荐）或作为同级兄弟插入

#### 样式设置

- **转换富文本语法**：启用后将源格式的高亮语法转换为 Orca 富文本格式
- **挖空模式**：启用后将指定语法转换为 Orca 挖空（遮挡）效果

### 挖空模式

挖空模式允许将特定语法标记的内容转换为 Orca 的挖空（遮挡）效果，常用于学习、记忆等场景。

#### 支持的挖空语法

| 语法 | 值 | 说明 |
|------|-----|------|
| `〖内容〗` | `tortoise` | 龟兔赛跑风格挖空 |
| `[[c1::xx]]` | `cloze-idx-bracket` | 带索引的方括号挖空（Anki 格式） |
| `{{c1::xx}}` | `cloze-idx-brace` | 带索引的花括号挖空 |
| `[[xx]]` | `bracket` | 方括号挖空 |
| `{{xx}}` | `brace` | 花括号挖空 |
| `**xx**` | `bold` | 粗体挖空 |
| `***xx***` | `bold-italic` | 粗体+斜体挖空 |
| `*xx*` | `italic` | 斜体挖空 |

**注意**：可多选语法，支持组合使用。`==高亮==` 语法由"转换富文本语法"功能处理，不在挖空模式中。

### 富文本转换

启用"转换富文本语法"后，插件会自动识别并转换以下富文本样式：

| 源格式 | 高亮语法 | 转换后效果 |
|--------|----------|------------|
| Logseq | `==文本==` | Orca 背景高亮（黄色） |
| Obsidian | `==文本==` | Orca 背景高亮（黄色） |
| SiYuan | `{{yellow背景色}}` | Orca 背景高亮（对应颜色） |
| Orca | 原生高亮 | 保持原样 |

支持的高亮颜色：红色、蓝色、绿色、黄色。

## 导出功能

### 导出格式

插件支持以下导出格式，根据选择的样式不同，导出选项会有所差异：

| 格式 | 说明 | 适用场景 |
|------|------|----------|
| **大纲（Logseq）** | Logseq 风格大纲格式 | 迁移到 Logseq |
| **大纲（Obsidian）** | Obsidian 风格大纲格式 | 迁移到 Obsidian |
| **Orca Note 样式** | Orca Note 原生风格 | 备份或跨设备同步 |
| **SiYuan 样式** | 思源笔记风格格式 | 迁移到思源笔记 |
| **Markdown** | 标准 Markdown 格式 | 通用文档交换 |
| **HTML（大纲）** | HTML 大纲格式 | 网页展示 |
| **HTML（正文）** | HTML 正文格式 | 网页嵌入 |
| **Word** | Microsoft Word 格式 | 打印或分享 |
| **JSON** | JSON 数据格式 | 程序接口或数据备份 |
| **OPML** | OPML 大纲格式 | 大纲交换（如思维导图） |
| **纯文本** | 纯文本格式 | 简单文本提取 |

### 导出样式

选择导出样式后，插件会按照对应笔记应用的语法规则进行转换：

| 样式 | 高亮语法 | 挖空语法（默认） | 列表格式 |
|------|----------|------------------|----------|
| **Logseq** | `==文本==` | `**文本**` | `- ` 无序列表 |
| **Obsidian** | `==文本==` | `**文本**` | `- ` 无序列表 |
| **Orca Note** | 原生高亮 | `**文本**` | 保持层级结构 |
| **SiYuan** | `{{color文本}}` | `**文本**` | `- ` 无序列表 |

### 导出设置

导出对话框提供以下设置选项：

#### 基础设置

- **最大深度**：导出时的层级深度（建议范围 2~10，默认为 6）
- **大纲模式**：选择大纲样式（大纲、有序列表、任务列表、层级大纲）

#### 样式设置

- **转换富文本语法**：启用后将 Orca 富文本格式转换为目标格式的语法
- **挖空模式**：启用后将 Orca 挖空效果转换为指定语法

#### Word 专用设置

- **Word 标签模式**：选择导出 Word 时使用的高亮标签
  - **传统标签（推荐）**：使用 `<mark>` 和 `<span style>` 标签，兼容性好
  - **Orca 原生标签**：使用 `<span class="orca-inline">` 标签，保留 Orca 特有样式

## 支持的格式详解

### 导入格式

| 格式 | 文件扩展名 | 说明 |
|------|------------|------|
| **Markdown** | `.md` | 标准 Markdown 文件，支持标题、列表、代码块、表格、链接、图片 |
| **纯文本** | `.txt` | 纯文本文件，按换行符分割为段落 |
| **OPML** | `.opml` | OPML 大纲文件，用于思维导图和大纲交换 |
| **JSON** | `.json` | JSON 数据文件，支持特定结构的笔记数据 |
| **XML** | `.xml` | XML 文件，支持特定结构的笔记数据 |

### 导出格式

#### 大纲格式

- **大纲**：标准无序列表格式
- **有序列表**：数字编号列表格式
- **任务列表**：带复选框的任务列表格式（`- [ ]`）
- **层级大纲**：缩进表示层级的纯文本格式

#### Markdown 格式

标准 Markdown 格式，包含：
- 标题（`#` 到 `######`）
- 列表（有序和无序）
- 代码块（```）
- 表格
- 链接和图片

#### HTML 格式

两种 HTML 格式：
- **HTML（大纲）**：使用 `<ul>/<li>` 结构的大纲格式
- **HTML（正文）**：使用 `<div>` 结构的正文格式，带样式

#### Word 格式

生成 Microsoft Word 兼容的 HTML 文件（`.doc`），包含：
- 层级缩进（通过 `margin-left`）
- 表格支持
- 高亮样式（通过 `<mark>` 或 `<span>`）

#### JSON 格式

结构化 JSON 数据，包含：
- 节点层级关系
- 文本内容和富文本片段
- 元数据

#### OPML 格式

标准 OPML 2.0 格式，用于大纲交换，可导入到思维导图工具。

## 常见问题

### Q1：导入后图片无法显示？

A：插件使用 Orca 的原生图片解析功能。如果图片无法显示，请检查：
- 图片链接是否有效
- 图片 URL 是否包含特殊字符（如括号）
- 是否在网络环境中（在线图片需要网络连接）

### Q2：导入后表格格式错乱？

A：Orca 的表格结构与 Markdown 不同。导入时插件会尽力保持表格结构，但复杂表格可能需要手动调整。

### Q3：挖空效果不生效？

A：请检查：
- 是否在设置中启用了"挖空模式"
- 是否选择了正确的挖空语法
- 语法是否正确（如 `〖内容〗` 使用的是中文方括号，不是 ASCII 字符）

### Q4：导出的 Word 文件样式不正确？

A：请尝试切换"Word 标签模式"：
- 传统标签：兼容性更好，适合大多数情况
- Orca 原生标签：保留更多 Orca 特有样式，但可能在某些 Word 版本中显示异常

### Q5：如何取消正在进行的导入/导出？

A：在进度浮层中点击"取消"按钮，或按 `Esc` 键。插件会立即停止操作并清理已创建的内容。

### Q6：导入时提示"skipped children of failed node"？

A：在 v2.5.1 中已修复。若仍出现：
- 确认使用的是 v2.5.1 或更高版本
- 打开"调试开关"查看 `[Import]` 前缀的详细日志
- 检查导入文件是否包含特殊格式导致解析失败
